cetaceanspeak: whale_speaker.py — response synthesizer (both modes, all species)

whale_speaker.py (~500 lines):

  CetaceanSynthesizer — token list → species-appropriate WAV
    humpback:    frequency sweeps + vibrato; _vibrato_sweep, _harmonic
    sperm_whale: click trains; a-coda (AREV/800Hz) vs i-coda (AFWD/2500Hz)
                 diphthong as FSPLIT (interleaved trains) + FFUSE (converging)
                 signature coda (IFIX) = short-long-short ICI pattern
    orca:        multi-harmonic burst-pulse via _burst_pulse
    All species: amplitude envelopes, -18 dBFS normalization, CLINK=no-gap

  ResponsePlanner — incoming expression → (response_name, token_list)
    15-entry strategy table covering all HUMAN_EXPRESSIONS archetypes
    Per-species token sequences: preserves structural register, mirrors
    phonological contrasts, escalates song/narrative responses

  WhaleSpeaker — unified entry point
    speak(expression, species, output_path) → WAV  (manual mode)
    respond(wav_path, species, output_path) → dict (auto-response mode)
      pipeline: WAV → whale_audio.translate_wav → structural signature
               → ResponsePlanner → CetaceanSynthesizer → WAV

  CLI:
    --expression / -e  EXPR    manual mode
    --respond    / -r  WAV     respond to incoming WAV
    --species    / -s  SPECIES humpback | sperm_whale | orca
    --output     / -o  PATH    output WAV path
    --list       / -l          list available expressions and species
    --quiet      / -q          suppress analysis output

pyproject.toml:
